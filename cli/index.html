    CLI reference¶

    Every toolr subcommand documented in one place. Top-level flags first, then each subcommand under its namespace. Each section is anchored for grepping.

    Top-level¶

    toolr --help

    In-project CLI tooling support

Usage: toolr [OPTIONS] [COMMAND]

Commands:
  example  Example commands
  project  Operations on the current repo's tools/ directory
  self     Operations on toolr itself

Options:
  -h, --help     Print help
  -V, --version  Print version

Output Options:
  -d, --debug
          Increase verbosity (also enables `DEBUG` logging).
  -q, --quiet
          Suppress non-error output.
      --timestamps
          Prepend ISO-8601 timestamps to log lines.
      --no-timestamps
          Suppress log-line timestamps (default; overrides `--timestamps`).
      --timeout-secs <SECONDS>
          Default timeout applied to every `ctx.run(...)` subprocess (per-call `timeout_secs=` wins
          when set).
      --no-output-timeout-secs <SECONDS>
          Default no-output watchdog applied to every `ctx.run(...)` subprocess — abort if no
          stdout/stderr for this many seconds. Per-call `no_output_timeout_secs=` wins when set.
    • toolr --version — print the binary's version.
    • toolr --help — print top-level help. On a leaf command the long form prints the full docstring (rendered via termimad); -h prints the same content as a one-line summary.

    Output Options¶

    Five root-level flags that tweak how toolr's own output renders and the defaults for any ctx.run(...) subprocess your command starts. They're root-only — place them before the subcommand (toolr -d --timeout-secs 30 ci hello, not toolr ci hello -d --timeout-secs 30).

    Flag What it does
    -d / --debug Increase verbosity. Also enables Python DEBUG logging in the runner. Mutually exclusive with --quiet.
    -q / --quiet Suppress non-error output.
    --timestamps / --ts Prepend ISO-8601 timestamps to log lines.
    --no-timestamps / --nts Suppress log-line timestamps (default; wins over --timestamps).
    --timeout-secs SECONDS / --timeout Default timeout passed to every ctx.run(...) call. Per-call timeout_secs= still wins.
    --no-output-timeout-secs SECONDS / --nots Default "no output for N seconds" watchdog applied to every ctx.run(...). Per-call no_output_timeout_secs= still wins.

    How the timeouts interact with ctx.run:

    @command(group="example")
def slow(ctx):
    """Tools-author writes this, oblivious to root flags."""
    ctx.run("sleep", "30")                       # uses --timeout-secs default if set
    ctx.run("sleep", "30", timeout_secs=60)      # per-call wins; ignores root flag

    The defaults flow through the runner spec as ContextSpec.default_timeout_secs / default_no_output_timeout_secs. They're plain numbers in seconds; None (the default-default) means "no watchdog unless the caller opts in."

    toolr project ...¶

    Operations on the current repo's tools/ directory.

    toolr project init¶

    Scaffold tools/ in the current directory.

    Usage: toolr project init [--force] [--no-sync] [--venv-location {cache,in-tree}] [--no-example] [--python <version>] [--quiet]

    Flags:

    • --force — overwrite an existing tools/ directory.
    • --no-sync — skip the automatic uv sync after scaffolding.
    • --venv-location — cache (default) or in-tree. Sets the [tool.toolr] venv-location value in the generated pyproject.toml.
    • --no-example — skip generating tools/example.py.
    • --python — requires-python value for tools/pyproject.toml (defaults to the running Python's >=major.minor).
    • --quiet — suppress informational output.
    toolr project init --help

    Scaffold tools/ in the current directory

Usage: toolr project init [OPTIONS]

Options:
      --force                     Overwrite an existing tools/ directory
      --no-sync                   Skip the automatic `uv sync` after scaffolding
      --venv-location <LOCATION>  Where the tools venv should live [default: cache] [possible
                                  values: cache, in-tree]
      --no-example                Skip generating tools/example.py
      --python <VERSION>          `requires-python` value for tools/pyproject.toml (defaults to the
                                  running Python's >=major.minor)
  -y, --yes                       Auto-approve overwriting any conflicting scaffold files
  -q, --quiet                     Suppress informational output
  -h, --help                      Print help

    See also: Quickstart, Project configuration.

    toolr project deps sync¶

    Materialise (or refresh) the tools venv via uv sync.

    toolr project deps sync --help

    Run `uv sync` against tools/

Usage: toolr project deps sync

Options:
  -h, --help  Print help

    Runs the full sync flow (see Project configuration → sync interaction).

    toolr project venv path¶

    Print the absolute path to the resolved tools venv. Useful for shell-scripting or sanity-checking venv-location.

    toolr project venv path --help

    Print the absolute path to the tools venv

Usage: toolr project venv path

Options:
  -h, --help  Print help

    toolr project venv shell¶

    Spawn a subshell with the tools venv activated. $VIRTUAL_ENV is set, $PATH is prepended with the venv's bin/, and $TOOLR_VENV carries the venv path for prompt customisation.

    toolr project venv shell --help

    Spawn a subshell with the tools venv activated

Usage: toolr project venv shell

Options:
  -h, --help  Print help

    toolr project manifest rebuild¶

    Regenerate the static + dynamic manifest in place. Equivalent to what toolr does automatically when it detects drift, but explicit and reportable.

    toolr project manifest rebuild --help

    Regenerate the static + dynamic manifest in place

Usage: toolr project manifest rebuild

Options:
  -h, --help  Print help

    toolr self ...¶

    Operations on toolr itself (not on the project).

    toolr self completion print <shell>¶

    Print the completion script for <shell> to stdout. Useful for piping into your own dotfiles management.

    <shell>: bash, zsh, or fish.

    toolr self completion print --help

    Print the completion script for a shell to stdout

Usage: toolr self completion print <shell>

Arguments:
  <shell>  Shell to emit a completion script for [possible values: bash, zsh, fish]

Options:
  -h, --help  Print help

    toolr self completion install <shell> [--force]¶

    Write the completion script for <shell> into its standard location (~/.bash_completion.d/, ~/.zsh/completions/, or ~/.config/fish/completions/). --force overwrites an existing file without prompting.

    toolr self completion install --help

    Install the completion script for a shell into its standard location

Usage: toolr self completion install [OPTIONS] <shell>

Arguments:
  <shell>  Shell to install the completion script for [possible values: bash, zsh, fish]

Options:
      --force  Overwrite an existing differing file without prompting
  -h, --help   Print help

    toolr self cache list¶

    Tabular listing of every cached per-repo venv: source repo, size, last-used timestamp.

    toolr self cache list --help

    List every cached virtualenv with size and last-use timestamp

Usage: toolr self cache list

Options:
  -h, --help  Print help

    toolr self cache prune¶

    Remove orphan and stale cache entries.

    Flags:

    • --all — remove every cache entry (with confirmation).
    • --yes / -y — skip the confirmation prompt when used with --all.
    • --dry-run — show what would be deleted without deleting.
    • --stale-after-days <DAYS> — override the default staleness threshold (30 days).
    toolr self cache prune --help

    Remove orphan and stale cache entries

Usage: toolr self cache prune [OPTIONS]

Options:
      --all                      Remove every cache entry
      --stale-after-days <DAYS>  Override the staleness threshold [default: 30]
      --dry-run                  Show what would be deleted without deleting
  -y, --yes                      Skip confirmation when used with --all
  -h, --help                     Print help

    toolr self build-manifest <package>¶

    Generate a toolr-manifest.json fragment for a third-party package by AST-walking the installed package source. Pure Rust — no Python subprocess, no pip install -e . required.

    Flags:

    • <package> — looked up in the project's tools venv. Mutually exclusive with --source-dir.
    • --source-dir PATH — point the tool at a package's source tree directly (bypasses the venv lookup). Requires --package if the leaf directory name isn't the desired package name.
    • --package PKG — package name to embed in the fragment when using --source-dir.
    • --check — verify the on-disk manifest matches what regeneration would produce; exit 2 with a unified diff on drift.
    • --output PATH — write to a specific file instead of the package's default location.
    • --schema-version N — pin the emitted toolr_schema_version.
    toolr self build-manifest --help

    Generate a third-party manifest fragment for a package

Usage: toolr self build-manifest [OPTIONS] [PACKAGE]

Arguments:
  [PACKAGE]  Dotted Python package name to introspect (looked up in the tools venv)

Options:
      --source-dir <PATH>   Path to the package's source tree (bypasses the tools-venv lookup)
      --package <PKG>       Package name to embed in the fragment when using --source-dir (defaults
                            to the leaf directory name)
      --output <PATH>       Override the output path
      --schema-version <N>  Pin the emitted schema version
      --check               Verify the on-disk manifest matches regeneration
  -h, --help                Print help

    See also: Third-party packages.

    Internal subcommands¶

    Used by other tooling (shell completion scripts, CI). Not part of the user-facing surface; documented here for completeness.

    toolr __complete <cwd> <args...>¶

    Completion endpoint called by the installed shell scripts. Reads the current repo's manifest, prefix-matches against subcommands and arg values, writes candidates to stdout. Sub-50ms.

    toolr __build-static-manifest¶

    Regenerate only the static layer of the manifest (no Python introspection). Faster than toolr project manifest rebuild but won't pick up dynamically-registered commands.

    toolr __install-uv-now¶

    Force-install a managed copy of uv into $XDG_DATA_HOME/toolr/bin/, bypassing the usual consent prompt. Used by the install scripts and by toolr project deps sync after a consent flow.
